# Audio Player

> Современный аудиоплеер с интерфейсом на SVG, построенный на PyQt6.

Audio Player — это десктопное приложение, которое сочетает нативные возможности PyQt6 и веб-технологии. Интерфейс реализован в виде масштабируемого SVG-изображения и отображается встроенным браузерным движком Chromium. Программа поддерживает основные функции управления воспроизведением, плейлист, перемотку и масштабирование окна.


##  Стек технологий

| Компонент | Назначение |
|-----------|------------|
| [Python](https://www.python.org/) 3.6+ | Язык программирования |
| [PyQt6](https://riverbankcomputing.com/software/pyqt/) | Фреймворк для создания GUI |
| QtWebEngineView | Отображение веб-контента (HTML, SVG, JavaScript) |
| QWebChannel | Двусторонняя связь Python ↔ JavaScript |
| QMediaPlayer / QAudioOutput | Воспроизведение аудиофайлов |
| SVG | Разметка графического интерфейса |


##  Возможности

- **Воспроизведение аудиофайлов** форматов `.mp3`, `.wav`, `.ogg`, `.m4a`, `.flac`
- **Плейлист** — выбор нескольких файлов через диалог
- **Управление воспроизведением**: play/pause, stop, перемотка ±5 сек, начало/конец трека
- **Автопереключение** на следующий трек при завершении текущего
- **Отображение** имени текущего трека и таймера (`MM:SS / MM:SS`)
- **Прозрачное frameless-окно** без стандартных рамок, всегда поверх остальных
- **Масштабирование интерфейса** кнопками «+» / «–»
- **Перетаскивание окна** за элементы с атрибутом `draggable="true"`
- **Условное отображение элементов** (кнопки Play/Pause меняются в зависимости от состояния плеера)
- **Отладочный режим** — подсветка кликнутых элементов и вывод логов в консоль


##  Требования

- **Операционная система**: Windows, macOS, Linux (протестировано на Windows)
- **Python**: 3.6 или выше
- **Библиотеки Python**:
  - `PyQt6` (включая модули `QtWebEngineWidgets`, `QtWebChannel`, `QtMultimedia`)
- **Файл интерфейса**: `new_testlayout.svg` в корневой директории


##  Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/audio-player.git
cd audio-player
```

### 2. Создание виртуального окружения (рекомендуется)

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install PyQt6
```

> **Примечание**: PyQt6 включает все необходимые модули (`QtWebEngineWidgets`, `QtWebChannel`, `QtMultimedia`). Дополнительные библиотеки не требуются.

### 4. Запуск

```bash
python main.py
```

##  Архитектура

Взаимодействие между слоями организовано двусторонне:

- **JavaScript → Python**: через `QWebChannel` и слоты, декорированные `@pyqtSlot` (класс `Bridge` в `main.py`)
- **Python → JavaScript**: выполнение JS-кода методом `runJavaScript()` (класс `LogicHandler` в `logic.py`)

**Схема обработки клика:**

1. Пользователь кликает по SVG-элементу.
2. JavaScript-обработчик формирует JSON с тегом и атрибутами элемента (кроме `class`).
3. JSON передаётся в Python через вызов `api.log()`.
4. `LogicHandler.handle_click()` парсит JSON, извлекает значение атрибута `idx` (например, `"play_pause"`) и вызывает соответствующий метод.
5. Метод изменяет состояние плеера и при необходимости обновляет интерфейс через `run_js()`.

##  Конфигурация

В файле `main.py` определены следующие константы:

```python
SVG_FILENAME = "new_testlayout.svg"  # Имя SVG-файла интерфейса
S = 6.5                               # Начальный масштаб
DEBUG = False                         # Отладочный режим
```

- **`DEBUG = True`** включает вывод JSON-данных в консоль и подсветку кликнутых элементов красным цветом (полезно при разработке SVG-макета).

##  Сборка в исполняемый файл

Для создания standalone `.exe` (Windows) можно использовать PyInstaller:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --add-data "new_testlayout.svg;." main.py
```

Исполняемый файл появится в папке `dist/`.
